Yes — in .NET, you can absolutely use compiled assembly files (.dll or .exe) together with their XML documentation files as input to DocFX to generate API documentation in Markdown (.md) format.
Here’s how it works and how to set it up:

How DocFX Uses Assemblies + XML Docs

Assembly (.dll or .exe) → DocFX uses reflection to read the public API surface.
XML Documentation (.xml) → Generated by the C# compiler when you enable XML documentation output in your project settings. This contains your <summary>, <remarks>, <param>, etc.
DocFX merges both sources so that the generated API docs include:

API signatures from the assembly
Your inline XML comments from the .xml file




Steps to Generate API .md Files from Assemblies + XML Docs
1️⃣ Enable XML Documentation in Your Project
In your .csproj:
Xml<PropertyGroup>
  <GenerateDocumentationFile>true</GenerateDocumentationFile>
  <NoWarn>1591</NoWarn> <!-- Optional: suppress missing XML comment warnings -->
</PropertyGroup>

This will produce a .xml file alongside your .dll in the bin folder.

2️⃣ Install DocFX
dotnet tool install -g docfx


3️⃣ Create a DocFX Project
docfx init 

NOTE: answer NO to PDF support  also replace "src" json as shown below

This creates a docfx.json config file.

4️⃣ Configure docfx.json to Use Assemblies + XML Docs
Example:
Json{
  "metadata": [
    {
      "src": [
        {
          "files": [ "bin/*.dll" ],
          "exclude": [ "**/obj/**" ]
        }
      ],
      "dest": "api",
      "includePrivateMembers": false
    }
  ],
  "build": {
    "content": [
      { "files": [ "api/**.yml" ] },
      { "files": [ "api/**.md" ] }
    ],
    "dest": "_site"
  }
}


Note: DocFX automatically looks for the .xml file with the same name and path as the .dll.


5️⃣ Generate Metadata (YAML) from Assemblies
docfx metadata

This step reads your .dll + .xml and produces .yml files in the api folder.

6️⃣ Build Markdown API Docs
docfx build

This converts .yml into .md (and optionally HTML if you want a website).

✅ Result: You’ll have .md files for each API type in your api folder, with your XML comments embedded.

Run docfx --serve to launch the site.
