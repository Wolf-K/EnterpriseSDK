In C#/.NET, the XML documentation member name format is defined by the C# compiler and the ECMA-334 / ECMA-335 standards.
When you convert a fully qualified type, property, or method name into an XML documentation member name (as found in .xml doc files generated by the compiler), you must follow specific rules.

1. General Format
The XML documentation member name always starts with a prefix letter indicating the kind of member:

Prefix	Meaning
T:	Type (class, struct, interface, enum, delegate)
F:	Field
P:	Property
M:	Method
E:	Event
N:	Namespace
Example:

Text

Copy code
T:MyNamespace.MyClass
P:MyNamespace.MyClass.MyProperty
M:MyNamespace.MyClass.MyMethod(System.String,System.Int32)
2. Fully Qualified Names
Use the full namespace and type name.
Nested types use a dot (.) between the outer and inner type names.
Generic types append `n where n is the number of generic parameters.
Example:

Text

Copy code
T:System.Collections.Generic.List`1
3. Method Signatures
Parameter types are fully qualified.
Use , to separate parameters.
Use [] for arrays.
Use @ for escaped identifiers (if needed).
Use & for ref or out parameters.
Use * for pointer types.
Use !!n for generic method parameters.
Use `n for generic type parameters.
Example:

Text

Copy code
M:MyNamespace.MyClass.MyMethod(System.String,System.Int32)
M:MyNamespace.MyClass.GenericMethod``1(``0)
M:MyNamespace.MyClass.RefMethod(System.Int32@)
M:MyNamespace.MyClass.PointerMethod(System.Int32*)
4. Properties and Indexers
Properties: P: followed by the fully qualified property name.
Indexers: Represented as Item with parameter list.
Example:

Text

Copy code
P:MyNamespace.MyClass.MyProperty
P:MyNamespace.MyClass.Item(System.Int32)
5. Operators and Special Names
Special operators have fixed names:

C# Operator	XML Doc Name
op_Addition	Addition
op_Equality	Equality
op_Implicit	Implicit
op_Explicit	Explicit
Example:

Text

Copy code
M:MyNamespace.MyClass.op_Addition(MyNamespace.MyClass,MyNamespace.MyClass)
6. Constructors and Destructors
Constructor: .ctor
Static constructor: .cctor
Destructor: Finalize
Example:

Text

Copy code
M:MyNamespace.MyClass.#ctor(System.String)
M:MyNamespace.MyClass.#cctor
7. Escaping Special Characters
Use \ before special characters in identifiers (rare in C# but possible in IL).
Generic parameters in methods: n for method generics, `n `` for type generics.
✅ Summary Rule:

XML documentation member names are deterministic strings generated by the compiler that encode the member kind, fully qualified name, and parameter types in a strict format. They are used for linking <see> and <paramref> tags in XML doc comments.
